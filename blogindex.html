T10 - Linked Lists

Think of an array. It stores a collection of data. Linked lists are similar because they also store a collection of data. They store elements on behalf of client code. An array gets one large chunk of memory and each element is partitioned a portion of that memory chunk. A linked list allocates memory for each individual element (also called a "node"). A linked list gets its structure by using pointers to connect all its nodes together like links in a chain.

T9 - Pointers

An understanding of pointers is essential for understanding most programming algorithms used commonly today. On the most basic level, pointers are variables. Pointers are references to memory blocks that contain a stored value. They are used to reference a value already stored in memory. They are not the value they are referencing. They are not the memory block they are referencing. The value they are referencing can change and pointers can reference a new memory block at any time.

T8 - Static vs. Dynamic Programming Languages

In a statically-typed programming language, every single variable name must be bound to a type - integer, string, etc. - (at compile time by explicit data declaration) and an object (which is optional; if not bound to an object, the variable name will be null). Variable names can only be reassigned to another object of the same type. C, Java, and C++ are all statically-typed languages.

T6 - Object-Oriented Programming vs. Functional Programming

OOP was popularized mainly because of its reusability. One programmer can start a project, creating classes and methods, and another programmer can finish the project without having to modify the original code very much (or at all). OOP is built off of the idea of blocks (objects) of data. Each object (piece of data) is characterized by the class it belongs to. Classes are like presets. They give their objects preset methods and pieces of data that can be used by those methods. OOP concepts are pretty easy to grasp because the real world is modelled this way. A "couch" is an object that could belong to the "chair" class.

T5 - A Beginner's Guide to Ruby Classes

Ruby classes are best explained by example. Let's pretend that you have a balloon. Your balloon has certain characteristics. All balloons have a color, a volume, and a weight (when empty). These three characteristics can be modeled by a class in Ruby. The color, volume, and weight of your balloon will be instance variables of the general balloon class. The general balloon class can be used to create several different types of balloons (with different colors, volumnes, and weights) for different situations.

C5 - Stereotype Threat

"Stereotype threat refers to being at risk of confirming, as self-characteristic, a negative stereotype about one's group" (Steele and Aronson, 1995) .

Read that sentence again. Stereotypes often hold some bit of truth for some people. However, when people know that they being identified by a stereotype, they come at risk of confirming this stereotype and identifying with it. Most studies on stereotype threat are concerned with African Americans and college test scores. The stereotype studied is usually that black students will under-perform white students in academic challenges.
