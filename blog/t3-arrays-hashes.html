Arrays / Hashes

5/17/15

While learning Ruby I have been puzzled by the differences between arrays and hashes. Why use one or the other? According to linuxtopia.com the difference is small, "Ruby's arrays and hashes are indexed collections. Both store collections of objects, accessible using a key. With arrays, the key is an integer, whereas hashes support any object as a key." But this doesn't tell us when to use one over the other.

Arrays are ordered, hashes are not. This means that we can sort an array and easily retrieve data from a specified location. This makes arrays better for data that may require reordering and data that we will want to keep track of given a specific array location.

Hashes are great for making psuedo-objects, fake objects to simulate user data with a list of specific characteristics. This is because we can make a hash with keys that resemble class variable names and give them any values we want. We can then access this data with these familiar variable names. This works as long as their order doesn't matter.
