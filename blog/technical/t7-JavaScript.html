T7 - Learning Ruby vs. Learning JavaScript

6/15/15

Disclaimer

After spending some time learning both Ruby and JavaScript (with a background in Java and C), I can say that I personally liked learning Ruby more. However, I have not built any live projects with either language. Being such, I can only talk about my personal experience learning each language and not about their general usefulness. I learned both languages through online tutorials and Dev Bootcamp. In general, JavaScript is used for giving web pages responsive functionality and Ruby is used to make APIs. (In general) JS = Frontend; Ruby = Backend.

Learning Ruby

I enjoyed learning Ruby. I started with Ruby Monk and spent a lot of time looking through the Ruby Docs. Ruby Monk made things clear and easy for me as a beginner Rubyist. I also attended meetups to help me get excited about the language and keep my excitement level up while learning it. I haven't explored the use of any useful frameworks like Rails, Lotus, etc. But my experience to-date with Ruby has been smooth. Things in Ruby made sense to me as I learned them. The whole time I was learning Ruby, it was like Matz was speaking to my soul saying, "I know you loved Java, but Ruby is even easier to write, nicer to read, and more pleasant to use." I encourage anyone who wants to learn how to program or just add a new language to their toolbox to start with Ruby.

Learning JavaScript

With my background in Java, JavaScript wasn't too difficult to pick up (some say JavaScript and Java are completely unrelated, which is mostly true, but their syntax is similar in many ways). While learning JavaScript, I spent my days going through the codecademy JavaScript course (which was recently added, check it out!) and the codeschool JS course. I really liked both, but the codeschool course was top-notch. Their video and cheat-sheet style was a perfect mix for me. Now I am learning JavaScript at Dev Bootcamp. I highly recommend JavaScript for new programmers, but for different reasons than Ruby. Ruby spoiled me. Writing Ruby, learning Ruby, and reading Ruby are all much nicer than doing the same in JavaScript. However, while learning JavaScript I was quickly able to do cool things that other people could see. I could alter web pages without them reloading and create animations and games on them. This kind of power does come with a price, though. In order to use JavaScript with web pages, I had to learn how to make and manipulate web pages. I had to learn HTML and CSS before my JavaScript skills had any real-world use.

Summary

If you are looking at a quick introduction to programming, I suggest going down the Ruby road. Ruby is great for making APIs for web application development, among many other things. However, if you are more interested in animating web pages and front-end development, it may be better to learn JavaScript first. Either way, learn them both. Only then will you be able to become a true full-stack developer. And even a little experience in both languages will help you work with different developers more effectively.
