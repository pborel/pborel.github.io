Pointers

7/4/15

What are Pointers?

An understanding of pointers is essential for understanding most programming algorithms used commonly today. On the most basic level, pointers are variables. Pointers are references to memory blocks that contain a stored value. They are used to reference a value already stored in memory. They are not the value they are referencing. They are not the memory block they are referencing. The value they are referencing can change and pointers can reference a new memory block at any time.

How Does This Help Me?

  1. They allow different sections of code to easily share data. This can also be accomplished by copying the data, but pointers allow you to do this with minimal code.
  2. They are the foundation for more complex "linked" data structures (linked lists, binary trees).
A quick example:
We have an int called num. We also have a pointer called numPtr. numPtr references the memory location of num. We can get the value of num through numPtr by using dereference.

Dereference

Dereference is an operation that collects the value of the pointee (the variable that the pointer points to). You can use the dereference operation if and ONLY if the pointer is pointing to a value (has a pointee). The vast majority of pointer bugs result from the dereference operation being called on a pointer without a pointee. A pointer must have a pointee assigned before a dereference operation will work.

Pointer Assignment

We assign pointers with the assignment operator (=): e.g. p2 = p1. This means that p2 now points to the same pointee that p1 pointed to at the time of assignment. p2 does not point to p1. p1 is not the pointee of p2. p2 points to the pointee of p1. When two pointers reference the same pointee we call it "sharing."

The Null Pointer

The value null is the same as the value 0 and can therefore be used as a boolean false. A null pointer is a pointer that "points to nothing." However, this is tricky. A pointer that is unitialized (it was never pointed to a value when it was created), is a bad pointer. Bad pointers are not null pointers and they can ruin your life (depending on the programming language you use - compiled languages are nicer to programmers who forget to initialize their pointers). Calling a dereference operator on a null pointer (or a bad pointer) will return a runtime error. Assigning a pointer (p2) to a null pointer (p1) will copy the null value of p1 to p2.

Copying

Sharing is when one pointer points to the same pointee of another pointer. This means that two points point to the same memory location - the same pointee. They "share" this pointee. This pointee is also called a "shallow copy." This is good because it is easier for the computer to point two pointers to the same memory location than it is to create a whole new variable to store the same value for the second pointer.

Copying (or "deep copy") is when one pointer copies the value of another pointer. A whole new value has to be stored in memory. This new value is identical to another value (which is redundant). This is good because each pointer has their own unique copy of the value. p1's pointee's value can change without affecting p2's pointee's value.

It is easier/quicker to share a value between two pointers than to make a new copy of the memory location to be referenced by a second pointer. However, sharing is dangerous. If p2 deletes or alters the shared value, p1 suffers the consequences of having a deleted or altered pointee.

How Does All of This Work?

Think of it like this: 1) the pointee must exist (have memory allocated to it - except in the case of a null pointer), 2) the pointer must exist (have memory allocated to it), 3) the pointer must point to the pointee (or else it's a bad pointer and your life will be ruined).
