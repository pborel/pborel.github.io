T11 - Searching Algorithms (Binary and Linear Search)

7/11/15

What is a Searching Algorithm?

A searching algorithm is used to search through a data structure for a specific value or set of values. Google uses a complex searching algorithm to find and rank websites.

Binary Search

Binary search searches through a data set by comparing the middle value to the input value. If the input value is greater than the middle value, it looks at the upper half of the data set and repeats its search. If the input value is less than the middle value, it looks at the lower half of the data set and repeats its search. Because of this, binary search requires the input data to be sorted. Binary search is considered to be one of the most cost-effective searching algorithms.
  1. The input data set must be sorted.
  2. Binary search requires random access to the input data set.
  3. The cost of binary search is O(log n).
def binary_search(num, sorted_array, min=0, max=(sorted_array.length-1))
  middle = (max + min) / 2

  return -1 if min > max

  if num < sorted_array[middle]
    binary_search(num, sorted_array, min, (middle - 1))
  elsif num > sorted_array[middle]
    binary_search(num, sorted_array, (middle + 1), max)
  else
    middle
  end
end

Linear Search

Linear search (also called sequential search) searches through a set of data one by one starting from the beginning and ending once the wanted value is found.
  1. The set of data does not have to be sorted (linear search can stream data of arbitrary size).
  2. Linear search requires equality comparisons for the input data set (like an array or a hash).
  3. The cost of linear search is O(n).
def linear_search(obj, arr)
  index = 0
  while arr.length >= index
    if arr[index] != obj
      index += 1
    else
      return index
    end
  end
  return nil
end

def global_linear_search(obj, arr)
  answer = []
  index = 0
  while arr.length >= index
    if arr[index] != obj
      index += 1
    else
      answer << index
      index += 1
    end
  end
  return answer.length > 0 ? answer : nil
end

Differences Between the Two

  • Binary search requires the input data to be sorted; linear search does not - therefore, linear search can stream data of arbitrary size.
  • Binary search requires an ordering comparison; linear search only requires equality comparisons.
  • The cost of binary search is O(log n); the cost of linear search is O(n).

Benchmarking in Ruby

n = 500
array = (0..n).to_a

Benchmark.bm do |bm|

  bm.report do
    n.times do
      index(n, array)
    end
  end

  bm.report do
    n.times do
      binary_search(n, array)
    end
  end

  bm.report do
    n.times do
      linear_search(n, array)
    end
  end

  bm.report do
    n.times do
      global_linear_search(n, array)
    end
  end
end
