T11 - Searching Algorithms

Binary search searches through a data set by comparing the middle value to the input value. If the input value is greater than the middle value, it looks at the upper half of the data set and repeats its search. If the input value is less than the middle value, it looks at the lower half of the data set and repeats its search. Because of this, binary search requires the input data to be sorted. Binary search is considered to be one of the most cost-effective searching algorithms.

T10 - Linked Lists

Think of an array. It stores a collection of data. Linked lists are similar because they also store a collection of data. They store elements on behalf of client code. An array gets one large chunk of memory and each element is partitioned a portion of that memory chunk. A linked list allocates memory for each individual element (also called a "node"). A linked list gets its structure by using pointers to connect all its nodes together like links in a chain.

T9 - Pointers

An understanding of pointers is essential for understanding most programming algorithms used commonly today. On the most basic level, pointers are variables. Pointers are references to memory blocks that contain a stored value. They are used to reference a value already stored in memory. They are not the value they are referencing. They are not the memory block they are referencing. The value they are referencing can change and pointers can reference a new memory block at any time.
